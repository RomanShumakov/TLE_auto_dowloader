# TLE Auto Downloader

Скрипт на Python для автоматического скачивания и структурирования данных TLE (Two-Line Element) с портала Space-Track.org.

## Описание
Программа авторизуется на сервисе Space-Track, запрашивает актуальные данные о спутниках (General Perturbations) за последние 30 дней в JSON-формате,
преобразовывает и сохраняет их в TXT-файле.

## Основные функции
- **Авторизация через сессию**: Безопасный вход с использованием переменных окружения.
- **Умная фильтрация**: Запрос оптимизирован для получения актуальных данных (за последние 30 дней) с сортировкой по ID и времени.
- **Структурированное хранение**: Файлы автоматически распределяются по папкам вида `TLE/YYYYMMDD/`.
- **Проверка целостности**: Скрипт проверяет количество записанных строк, чтобы исключить потерю данных при записи.

## Требования
Для работы скрипта требуются следующие библиотеки:
- `requests` — для работы с API.
- `python-dotenv` — для безопасного хранения учетных данных.

## Установка и настройка

1. **Клонируйте репозиторий**:
   ```bash
   git clone https://github.com/твой-профиль/tle-auto-downloader.git
   cd tle-auto-downloader
   ```

2. **Установите зависимости через Poetry**:
   Убедитесь, что у вас установлена [Poetry](https://python-poetry.org/docs/#installation). Затем выполните:
   ```bash
   poetry install
   ```
   *Эта команда автоматически создаст виртуальное окружение и установит библиотеки `requests` и `python-dotenv`.*

3. **Настройте переменные окружения**:
   Создайте в корневой папке проекта файл `.env` и добавьте в него свои учетные данные от Space-Track.org:
   ```env
   LOGIN=ваша_почта@example.com
   PASSWORD=ваш_пароль
   ```

## Использование
Для запуска скрипта используйте команду:
```bash
poetry run python main.py
```

После выполнения появится файл ALL.txt по адресу: папка проекта -> папка "TLE" -> папка с текущей датой. Каждая запись в файле состоит из 3-х строк:
1. Имя объекта
2. TLE Line 1
3. TLE Line 2

## Технические детали
### Основной рабочий эндпоинт:

https://www.space-track.org/basicspacedata/query/class/gp/EPOCH/%3Enow-30/orderby/NORAD_CAT_ID,EPOCH/format/json
-   Описание: Запрашивает данные General Perturbations (GP, самый полный и детализированный набор орбитальных параметров спутника) за последние 30 дней.
-   Сортировка: Сначала по номеру спутника (NORAD_CAT_ID), затем по времени (EPOCH). Это позволяет сгруппировать все обновления для каждого конкретного объекта.
-   Формат: JSON.

### Альтернативные эндпоинты

1. История обновлений (полный каталог)

https://www.space-track.org/basicspacedata/query/class/gp/orderby/EPOCH%20desc/format/json
-   Запрашивает последние записи для всех объектов в базе без ограничения по времени. Удобно для получения актуального состояния всей группировки
(неактивные спутники учитываются). Сортировка от новых к старым.

2. Низкоорбитальные объекты (LEO) в формате 3LE

https://www.space-track.org/basicspacedata/query/class/gp/EPOCH/>now-30/MEAN_MOTION/>11.25/format/3le
-   Фильтрация: Выбирает объекты с количеством оборотов более 11.25 в сутки (низкая околоземная орбита).
-   Особенности: Возвращает данные в текстовом стандарте 3LE (не JSON). Подходит для загрузки в специализированное ПО для расчетов.

3. Проверка метаданных (SatCat)

https://www.space-track.org/basicspacedata/query/class/satcat/limit/1/format/json
-   Запрос к каталогу спутников (Satellite Catalog). Содержит общую информацию: дату запуска, статус объекта, страну-владельца.
Лимит в 1 спутник (limit/1) используется для проверки связи с API или тестирования структуры ответа.
